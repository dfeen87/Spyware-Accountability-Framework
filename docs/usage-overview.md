# Usage Overview

The Spyware Accountability Framework provides three core pipelines to analyze and map mercenary spyware infrastructure and ecosystems. This document outlines how to run these pipelines locally using the provided synthetic examples.

## Prerequisites

Ensure you have installed the framework in a suitable Python environment (Python 3.9+).

```bash
git clone https://github.com/dfeen87/Spyware-Accountability-Framework.git
cd Spyware-Accountability-Framework
pip install -e .[dev]
```

## Running the Pipelines

The framework relies on the `pipelines/` directory. Each pipeline is a standalone module that can be executed from the command line, taking input data and configuration, passing it through the AILEE trust layer, and generating actionable, defensive outputs.

### 1. Network Forensics Pipeline

This pipeline analyzes network artifacts (e.g., extracted flow features, TLS fingerprints, DNS resolutions) to detect spyware-like behavior, such as C2 beaconing or anomalous domain resolutions.

**Input:** A JSON or Markdown description of network features (see `examples/synthetic_network_capture_description.md`).
**Output:** A JSON report containing risk scores, AILEE classifications, and extracted IOCs.

```bash
python -m pipelines.network_forensics_pipeline \
    --input examples/synthetic_network_capture_description.md \
    --output /tmp/network_report.json
```

### 2. OSINT Vendor Mapping Pipeline

This pipeline ingests structured Open Source Intelligence (OSINT) regarding corporate entities, hosting providers, jurisdictions, and known domains to build a relational graph of a mercenary spyware vendor's ecosystem.

**Input:** A structured JSON dataset of vendor entities and relationships (see `examples/synthetic_osint_dataset.json`).
**Output:** A JSON or GraphML representation of the ecosystem, scored for risk by the AILEE layer, and a summary report.

```bash
python -m pipelines.osint_vendor_mapping_pipeline \
    --input examples/synthetic_osint_dataset.json \
    --output /tmp/osint_graph.json
```

### 3. Reporting Pipeline

The Reporting Pipeline aggregates the outputs from the Network and OSINT pipelines to generate comprehensive, human-readable briefs (Markdown) and machine-readable artifacts (IOC lists, defensive rules).

**Input:** The outputs generated by the previous pipelines (`/tmp/network_report.json`, `/tmp/osint_graph.json`).
**Output:** A Markdown brief and a JSON list of verified IOCs.

```bash
python -m pipelines.reporting_pipeline \
    --network-report /tmp/network_report.json \
    --osint-graph /tmp/osint_graph.json \
    --output-dir /tmp/reports/
```
