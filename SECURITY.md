# Security Policy

## Supported Versions

At any given time, only the `main` branch and the latest tagged release receive security updates.

## Reporting a Vulnerability

If you discover a security vulnerability or note that a piece of synthetic data or rule example could be misused or could inadvertently target individuals, please **DO NOT** create a public issue.

Instead, please reach out with notice to dfeen87@gmail.com. 

We commit to:
1. Acknowledging your report within 48 hours.
2. Assessing the issue and providing an estimated timeline for a fix.
3. Keeping you informed of our progress.
4. Fully credit you (if desired) once the issue is resolved.

## Specific Abuse Vectors

Given the nature of the Spyware Accountability Framework, we explicitly ask you to report:
- Any real Personally Identifiable Information (PII) of victims or targets found in our datasets.
- Any real, functional exploit code or offensive tooling that violates our defensive-only mandate.
- Any use of this framework that targets individuals rather than infrastructure, vendors, or broad patterns.

See [Governance/Abuse Handling Policy](governance/abuse-handling-policy.md) for more details.
